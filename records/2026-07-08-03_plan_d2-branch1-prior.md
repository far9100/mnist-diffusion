# D2：分支一先驗登記（C1 揭盲前）

## Goal

依執行指令 `records/2026-07-08-02` §1.3 與定稿計畫 `records/2026-07-06-05` §7 D2，於 C1（DINOv2
which-FID 分離裁決，屬 P1/β 之揭盲事件）發生前，先行登記 CIFAR-100 四分支決策樹（`records/2026-07-06-05`
§7 D1）中分支一的整體先驗。此登記之唯一目的是把先驗釘死在資料揭盲前，避免落地後回填一個對己有利的
機率，屬「希望測試」（`records/2026-07-06-05` §11）之事前護欄。

四分支樹（D1，複述以定義分支一，不改口徑）：

1. 分離出現 且 CaF 勝 FID-min → 邊界條件復活（分支一）。
2. 分離出現 但 CaF 敗 FID-min → thesis 活、selector 死。
3. 不分離 但機制複製 → 診斷論文。
4. 皆否 → 負結果短文。

## Result

登記如下，並附兩則明文限定：

- **分支一整體先驗：約一成（≈10%）。** 此為分支一「分離出現 且 CaF 於 matched-budget 勝 FID-min」兩事件
  同時成立的整體主觀機率，照 `records/2026-07-06-05` §7 D2 原文之數量級（「整體約一成」）登記。
- **限定 (i)：誠實聲明、非裁決輸入。** 本先驗僅供資源配置與準備權重之用，不進入任何裁決條文；分支路由
  一律由 D1 之客觀觀察量（分離判定＋matched-budget 對決）決定，不由本機率決定（`records/2026-07-06-03`
  §11 兩護欄、`records/2026-07-06-05` §11）。
- **限定 (ii)：登記時點在 C1 揭盲前。** 本檔落於 P1/β 之 DINOv2 per-config FD-DINOv2 尚未計算、C1 三 seed
  分離裁決尚未揭盲之時（P0 尚未 PASS、P1 未 greenlight，見 `records/2026-07-08-02` §0.2、§2.4）。登記在先
  使此先驗不受後續 CIFAR-10 C1 結果污染。

先驗偏低（約一成）與計畫既有的誠實定位一致：CIFAR-10 confirmatory 上 FID-min 已勝 CaF、且存在結構性
Pareto 失明（`records/2026-07-06-05` §1、`records/2026-07-06-16` C8 引理），故「CIFAR-100 上分離重現 且
CaF 反轉勝過 FID-min」兩件事同時成立本屬小機率。此判斷為主觀先驗，非任何資料之外推。
